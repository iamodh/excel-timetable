# 프로젝트 배경: 왜 만들었나

## 기존 흐름

**관리자가 시간표를 업데이트할 때**

1. 구글 시트에서 시간표 수정
2. 수정된 화면을 이미지로 캡처
3. 카카오톡 단톡방에 이미지 공유 + 변경 공지

**학생이 시간표를 확인할 때**

1. 카카오톡 접속
2. 단톡방을 스크롤해서 시간표 이미지를 찾음
3. "이게 최신 버전이 맞나?" 스스로 판단

## 새 흐름

**관리자**

1. 구글 시트 수정
2. 관리자 페이지에서 **"시간표 최신화"** 버튼 클릭
3. 안내가 필요하면 **"공지 작성"** 버튼 클릭

**학생**

1. 웹 주소 접속 → PIN 입력 → 항상 최신 시간표

> 한 줄 요약: 시간표를 웹에 띄워서 **카카오톡 의존성을 제거**했습니다.

---

## 개선 효과

### 1. 관리자 업무 부담 감소

단순히 작업 단계가 3 → 2로 줄어든 것이 아닙니다. 진짜 효과는 **"캡처 → 공유" 루프 자체가 사라진 것**입니다.

- **기존:** 시트 수정 후 오타 하나만 발견해도 → 다시 캡처 → 다시 카톡 공유 → 이전 이미지와 섞여서 혼동되지 않도록 추가 설명까지 필요
- **지금:** 시트 수정 + 최신화 버튼 클릭 → 끝

수정 비용이 낮아지면 수정 빈도가 올라갑니다. 관리자가 "이 정도 사소한 수정은 그냥 다음에 하지 뭐"라고 미루던 작은 오류들도 바로 고칠 수 있게 됩니다. **결과적으로 학생들이 보는 시간표의 정확도가 올라갑니다.**

### 2. 학생의 열람 경험 개선

기존 이미지 방식의 문제점:

- 카카오톡을 꼭 켜야 시간표를 볼 수 있음
- 단톡방을 스크롤해서 이미지를 찾아야 함
- **"이게 몇 주차 시간표지?"**, **"변경 반영된 최신 버전인가?"** 를 구분하기 어려움

지금은 **웹 주소 하나가 곧 "현재 시점의 정답 시간표"** 입니다. 학생이 최신 여부를 판단할 필요가 없습니다.

### 3. 실시간 반영

관리자가 "최신화" 버튼을 누르면 수 초 이내에 모든 학생이 같은 화면을 봅니다. 누구는 어제 이미지를, 누구는 오늘 이미지를 보는 상황이 사라집니다.

## 보안 설계

### 1. PIN 접근 제어

외부 사람(학생이 아닌 사람)이 주소만 알아도 시간표를 열람할 수 없습니다.

- 학생들에게만 PIN 공유
- 관리자가 PIN을 바꾸면 **기존 세션이 즉시 무효화** → 졸업·이탈 학생 접근을 끊을 수 있음

**약한 지점:** PIN이 학생 간 단톡방에서 공유되는 순간 외부 유출 가능성이 생깁니다. 다만 관리자가 언제든 새 PIN을 발급할 수 있으므로 구조적 결함은 아니며, 필요 시 PIN을 주기적으로 교체하면 해결됩니다.

### 2. 구글 시트 주소 비공개

원본 구글 시트의 주소(Sheet ID)는 **서버에만** 저장됩니다.

- 학생 브라우저로 전달되는 것은 **"오늘 3시에 요가 수업"** 같은 이미 가공된 내용뿐
- 브라우저 개발자 도구로 뒤져도 원본 시트 주소는 복원 불가
- 즉, 학생이 시트 원본에 직접 접근하거나 다른 사람에게 시트 링크를 넘겨줄 수 없음

**코드 레벨 확인 완료:** Sheet ID는 서버 환경변수(`GOOGLE_SHEET_ID`)로만 관리되며, 클라이언트로 내려가는 어떤 데이터에도 포함되지 않습니다 (`lib/sheets.ts`에서만 참조, 클라이언트 컴포넌트 접근 없음).
