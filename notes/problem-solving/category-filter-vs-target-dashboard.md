# M18 카테고리 필터 → 관리자 합계 대시보드

매니저가 처음 말한 요구("카테고리 클릭하면 해당 수업만 보이게")를 학생용 필터 UI로 받아 적었다가, 본질이 학생 UX가 아니라 **카테고리별 목표 시수 검증**임을 알게 되어 M18을 재설계한 기록.

---

## 1. 처음 들었던 요구

매니저: "카테고리 클릭하면 해당 카테고리 수업만 시간표에 보이게 해줄 수 있어?"

PLAN의 초안 M18: "범례 영역에서 카테고리 ↔ 색상 매핑을 파싱하고, 카테고리 버튼 클릭 시 해당 수업만 강조/표시한다." 학생 페이지에 토글 버튼을 두는 형태.

`parseCategories()`가 이미 `{name, color}` 배열을 추출하고 있었으므로 UI만 붙이면 끝나는 작업처럼 보였다.

---

## 2. 추가 질문에서 드러난 진짜 문제

매니저와 다시 얘기하면서 드러난 본질:

- 카테고리별 **목표 시수**가 있다 — 5회차 누적 16h 기준. 단, 지역맞춤·외부연계·자율은 목표 없음.
- 매니저는 **목표 대비 부족한 카테고리**를 빠르게 찾고 싶다 (현재 시간표가 다 안 채워진 상태).
- 카테고리 클릭은 단순 필터가 아니라, **부족한 카테고리가 시간표 어디에 배치돼 있는지 식별 → 빈 슬롯 찾아 시트에서 채움** 흐름의 시작점이다.

학생용 필터 UI로는 합계가 안 보여 검증이 불가능하다. 게다가 매니저는 학생 페이지를 보지 않는다.

---

## 3. 설계 전환

M18을 학생용 필터에서 **관리자 카테고리 합계 대시보드**(`/admin/categories`)로 재정의.

### 화면 구조

상단: 회차×카테고리 매트릭스 표

```
카테고리    1회 2회 3회 4회 5회 │ 합계 / 목표
─────────────────────────────────────────────
밀착상담     2   1   2   1   1  │  7 / 16  ⚠ -9
체험활동     2   2   2   2   2  │ 10 / 16  ⚠ -6
진로         4   3   3   3   3  │ 16 / 16  ✓
─── 목표 없음 ───
자율         1   0   1   1   0  │  3
지역맞춤     0   0   1   0   0  │  1
```

- 부족분 큰 순으로 정렬, 목표 없는 카테고리는 구분선 아래
- 행 클릭 = 단일 카테고리 선택 (다중 선택 미지원)

하단: 시간표 그리드 — 학생용 컴포넌트 재사용

- `highlightCategory` prop이 있으면 비매칭 셀을 회색 처리(테두리만 남김)
- 회차 탭 전환해도 강조 유지 (상태는 페이지 단에서 보유)

매니저 작업 흐름:

1. 표에서 부족한 카테고리 식별
2. 행 클릭 → 그리드에서 해당 카테고리 위치 강조
3. 회차 탭 돌며 빈 슬롯 확인 → 시트 수정

## 4. 목표 시수를 어디에 둘 것인가

세 옵션을 검토했다.

| 옵션 | 진실원 | 변경 비용 | 안전성 |
|------|--------|----------|--------|
| 시트에서 함께 파싱 | 시트 단일 | 시트 레이아웃 + 파서 확장 | 높음 |
| 코드 상수 + 이름 매칭 | 코드 | 1파일 추가 | 중 (시트 이름 바뀌면 매칭 깨짐) |
| 관리자 KV 입력 | 두 곳 | KV + UI | 낮음 (진실원 분리) |

처음에는 진실원 단일화를 위해 옵션 A(시트 파싱)를 추천했다. 하지만 매니저가 "16시간 고정"이라고 못 박았기 때문에 카테고리별로 다른 목표 값이 들어올 가능성이 없었다.

즉, **목표 시수가 가변이라는 가정 자체가 오버엔지니어링**이었다. 실제 구조는 단순했다:

- 목표 있는 카테고리: 모두 16h
- 목표 없는 카테고리: 지역맞춤, 외부연계, 자율 (3개 고정)

이 구조라면 "예외 목록"만 코드에 두면 끝난다.

```ts
// lib/categoryTargets.ts
const NO_TARGET_CATEGORIES = new Set(["지역맞춤", "외부연계", "자율"])
const DEFAULT_TARGET_HOURS = 16

export function getCategoryTargetHours(name: string): number | null {
  return NO_TARGET_CATEGORIES.has(name) ? null : DEFAULT_TARGET_HOURS
}
```

이 방식의 장점:

- 시트 레이아웃 변경 없음
- 파서 확장 없음
- 새 카테고리가 추가되면 자동으로 16h 목표 적용 (보통 원하는 동작)
- 예외 카테고리 이름이 바뀔 가능성만 신경 쓰면 됨 — 그마저도 매칭 실패 시 표에서 "목표 없음"으로 노출되어 발견됨

매니저가 카테고리별로 다른 목표를 원하는 시점이 오면 그때 옵션 A 또는 C로 확장한다 (YAGNI).

## 5. 그리드 컴포넌트 재사용

학생/관리자가 같은 그리드를 쓴다. 별도 컴포넌트를 만들지 않고 prop 하나로 모드를 전환:

```tsx
<TimetableGrid highlightCategory={selectedCategory} />
```

`highlightCategory`가 없으면 학생 페이지처럼 풀컬러, 있으면 비매칭 셀 회색 처리. 그리드의 본질(요일×시간 렌더링, 셀 병합)은 같고 강조는 표시 옵션일 뿐이다.

---

## 6. 결정에서 얻은 교훈

- **사용자가 말한 UI ≠ 사용자가 풀려는 문제.** 매니저가 말한 "카테고리 필터"를 그대로 받아 학생 페이지에 토글을 만들었다면 매니저 일은 여전히 안 풀렸을 것이다. "이 UI로 무엇을 하시려고요?"를 한 번 더 물어본 것이 설계 방향을 바꿨다.
- **데이터의 가변성을 가정하기 전에 실제 가변성을 확인하라.** 목표 시수를 시트·KV·코드 어디에 둘지 한참 검토했지만 매니저는 "16시간 고정"이라고 답했다. 가변 데이터로 다루는 모든 비용이 불필요했다.
- **컴포넌트는 prop 하나로 확장 가능한 만큼만 확장한다.** 관리자용 그리드를 새로 만들지 않고 `highlightCategory` prop 하나로 끝냈다.
